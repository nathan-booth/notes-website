Lesson 5: Object-Oriented Programming

OOP

Object-Oriented Programming (OOP) in Python refers to a class-based (hierarchical) system that organizes data structures. OOP is useful for increasing intuitive data organization, code reusability, and decomposing large programs into subprograms, which can be built directly. The decomposition process is similar to creating multiple procedural functions in Python to solve a complex computational problem.

Class inheritance functions identically in Python and CSS; it has the property of passing on its characteristics or behavior to child classes and instances. Classes share attributes that can be called repetitively. Subclasses have precedence over superclasses and instances have precedence over classes.

Classes and Objects

A class defines an archetype of an object. An object (or instance) of a class is a specific version of that class, like my dog Watson is an object of class Dog. A constructor is used to initialize a class; Python uses the __init__ function name within a class definition. __init__ defines the attributes of the class. Classes allow programmers to create new data types and can be re-used.

Methods and Objects

Instance variables exist within a particular instance of a class. Scott Adams, who belongs to the class Human, is assigned male to the instance variable gender. This is separate from the class variable gender, which allows for several possible designations, including male and female. Class variables are assigned at the class level of code, not the function level within a class.

Methods are functions associated with a particular class. Instance methods are functions within a class definition that control that class's behavior. Method overriding happens when subclass methods are executed over superclass methods of the same name, parameters, and return type.

Collections and Abstraction

A library is a large collection of related modules and code. Modules are collections of related functions and classes built for specific purposes. Modules are written in self-contained files that programs can call upon at need, similar to HTML documents calling CSS or JavaScript files to fulfill styling and dynamic function, respectively. Abstraction refers to how code details are hidden in order to manage the complexity of the code, be they Python or CSS classes or the HTML DOM. The details of methods are hidden to emphasize their functionality to users over their construction.
